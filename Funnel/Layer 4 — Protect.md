# Layer 4 — Protect

**Service:** [[NestShield Technology]]
**Division:** [[NestShield]]
**Revenue:** Install fee + monthly recurring monitoring

---

## Entry Triggers

- Handoff from [[Layer 3 — Advise]] (risk report recommends monitoring)
- Handoff from [[Layer 1 — Assess]] (inspection found immediate water risk, fast-tracked)
- Direct: property manager searching for leak detection
- Insurance referral: carrier requires or incentivizes monitoring
- After a loss: client just had a water event

## Step-by-Step SOP

### 1. Site Qualification

Confirm the building is a fit:
- Water supply configuration (domestic, fire, irrigation — entry points)
- Existing shutoff valves (location, condition, type)
- Building construction type (what fails when water is present)
- Occupancy pattern (24/7 vs. vacant nights/weekends)
- Internet/network availability
- Pipe materials and age
- Historical losses and near-misses

### 2. System Design

- Sensor placement map (mechanical rooms, water heaters, risers, elevator pits, unoccupied units)
- Automatic shutoff valve locations and sizing
- Freeze sensor locations (attics, exterior walls, unheated spaces, garages)
- Connectivity plan (WiFi, cellular, hardwired)
- Alert routing (who, what order, what hours)
- BAS integration if applicable

### 3. Propose and Price

- System design summary with sensor/valve count
- Installation scope and timeline
- One-time install cost
- Monthly monitoring cost
- ROI based on client's risk profile (use their [[Layer 3 — Advise|Layer 3]] data)
- Insurance premium impact
- Warranty and service terms

### 4. Install

- Pre-install coordination (access, scheduling, tenant notification)
- Install sensors and valves per design
- Connect to monitoring platform
- Test every sensor and valve — trigger each, confirm alerts
- Test shutoff — confirm auto-shutoff and water restore
- Document and photograph everything

### 5. Commission and Train

- Walk client through monitoring dashboard
- Show what an alert looks like
- Explain escalation path (alert → notification → auto-shutoff → manual response)
- Provide emergency contacts and procedures
- Confirm who acknowledges alerts
- Hand over system documentation

### 6. Monitor and Maintain (ongoing)

- 24/7 monitoring via platform
- Monthly health reports
- Quarterly sensor/valve checks
- Annual system review (batteries, valve testing, expansion recommendations)
- Immediate response coordination on events

## The Loop Back

Monitoring data feeds back into the [[Revenue Funnel]]:
- System degradation → new [[Layer 1 — Assess|inspection]]
- Approaching failure → [[Layer 2 — Plan|estimate for replacement]]
- Portfolio patterns → new [[Layer 3 — Advise|risk engagement]]
- Client acquires new building → back to [[Layer 1 — Assess]]
