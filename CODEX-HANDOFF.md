# Codex Handoff

## What This Is

NestShield business — 2 brands + 1 side hustle. 3 websites to build.

## Structure

```
sites/
  brm/              — Brown Risk Management. Inspections + consulting. One website.
  nestshield/        — NestShield IoT water risk management. nestshield.net. The product.
  estimating/        — Standalone freelance estimating. Separate site.
```

## The Funnel

BRM (inspections → consulting) → NestShield (IoT)

Inspections find risk. Consulting advises on risk. NestShield eliminates risk.

NestShield also has its own entry point: "Book an Audit" — a paid scoping visit for IoT products. This is NOT a BRM inspection.

## What's Built

- Content strategy and positioning for each brand (in READMEs)
- Funnel logic and handoff language

## What's NOT Built

- Website code (each folder needs HTML/framework)
- Domain setup (nestshield.net exists, BRM and estimating TBD)
- Visual design / branding
- Contact forms, scheduling, audit booking flow

## Rules

- BRM inspection report is OBJECTIVE — no opinions. Consulting is the upsell.
- Never pitch. Findings speak.
- NestShield is the endgame. Everything feeds it.
- Estimating is standalone — no forced cross-sell.
- Risk management is the identity, not technology.
